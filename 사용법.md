# 사용법 (코딩 몰라도 OK)

## 1. 여는 방법

가장 간단한 방법: `index.html` 파일을 **더블클릭**하면 기본 브라우저(크롬 등)로 바로 열립니다.

VSCode를 쓰고 있다면 더 편한 방법도 있습니다.
1. VSCode 왼쪽 확장(Extensions) 아이콘 클릭 → "Live Server" 검색 → 설치
2. `index.html`을 열어둔 상태에서 오른쪽 아래 "Go Live" 버튼 클릭
3. 자동으로 브라우저가 열리고, 이후 파일을 수정하면 화면이 자동으로 새로고침됩니다 (저장할 때마다 브라우저 다시 켤 필요 없음)

## 2. 화면 구성

- **📖 지표 사전 탭**: 지표 카드를 클릭하면 발표시간, 발표기관, 중요도, 단위, 설명, 2020년~현재 시계열 차트, 최근 발표 내역(컨센서스/실제치/이전치)이 나옵니다.
- **📅 발표 캘린더 탭**: "이전달 / 이번달 / 다음달" 버튼으로 이동하며 그 달 한 달치 발표 일정을 한눈에 확인합니다. 지표를 클릭하면 사전 탭과 똑같은 상세 설명(차트 포함)이 나옵니다.
- 상단 카테고리 버튼(성장/물가/고용/소비/투자/통화정책/무역)을 누르면 사전·캘린더 둘 다 그 카테고리만 걸러서 보여줍니다.

### 시계열 차트는 어떻게 만들어지나요?

지표 상세를 열면 "2020년 ~ 현재 추이" 차트가 나옵니다. 이 차트는 `data.js`의 `history`에 직접 입력된 최근 값(실선+점)과, 2020년부터 그 시점까지의 빈 구간을 채우는 **자동 생성 예시 데이터**로 구성됩니다. 회색 안내 문구가 그 사실을 알려줍니다. 실제 값으로 채우고 싶다면 `history` 배열에 과거 발표분을 하나씩 추가하면 되고, 추가한 시점까지는 차트에도 실제로 입력한 값이 그대로 반영됩니다. 파란 선은 실제치, 주황 점은 컨센서스(예상치)입니다.

## 2-1. 새 데이터는 이렇게 넣어주세요 (추천 방법)

`data-imports` 폴더에 엑셀 파일을 넣고, Claude에게 "새 데이터 반영해줘"라고 말씀해 주시면 자동으로 읽어서 대시보드에 반영합니다. 인포맥스·ECOS·블룸버그 등 어디서 내보낸 파일이든 형식 상관없이 그대로 넣으시면 됩니다. 채팅에 표를 복사-붙여넣기 하는 것보다 훨씬 빠르고 정확해요.

## 3. 데이터는 어디에 있나요?

모든 데이터는 `data.js` 파일 하나에 들어 있습니다. VSCode에서 이 파일을 열어 메모장처럼 수정하면 됩니다. 저장(Ctrl+S) 후 브라우저를 새로고침(F5)하면 바로 반영됩니다.

> ⚠️ 지금 들어있는 발표일·컨센서스·실제치 수치는 화면이 어떻게 보이는지 보여주기 위한 **예시(가짜) 데이터**입니다. 실제 사용하시려면 각 공식 발표기관 홈페이지나 신뢰할 수 있는 경제 캘린더를 보고 직접 값을 채워 넣어 주세요.

### 3-1. 새 지표(사전 항목) 추가하기

`data.js`를 열고 `const indicators = [` 로 시작하는 부분을 찾으세요. 대괄호 `[` 바로 다음 줄에 아래처럼 항목 하나를 복사해서 붙여넣고 내용만 바꾸면 됩니다.

```js
{
  id: "kr_new_indicator",              // 다른 지표와 겹치지 않는 영문 이름 (자유롭게 작성)
  name: "새 지표 이름",                  // 화면에 보이는 한글 이름
  nameEn: "New Indicator Name",        // 영문 이름
  country: "한국",                      // 국가
  category: "소비",                     // 성장/물가/고용/소비/투자/통화정책/무역 중 하나
  institution: "발표기관 이름",
  importance: "중",                     // 상/중/하 중 하나
  unit: "%",                           // 단위
  frequency: "매월",                    // 발표주기
  releasePattern: "매월 15일 무렵",      // 대략적인 발표 시기 설명
  description: "이 지표가 무엇을 뜻하는지 설명하는 문장.",
  hasConsensus: true,                   // 시장 컨센서스(예상치)가 존재하면 true, 없으면 false
  history: [
    { date: "2026-07-01", consensus: "1.0%", actual: "1.2%", previous: "0.8%" },
  ],
},
```

- 항목 끝에 콤마(,)를 꼭 붙여야 다음 항목과 구분됩니다.
- `hasConsensus: false`인 지표(의사록 공개처럼 숫자로 딱 떨어지지 않는 이벤트)는 `history`의 `consensus`, `previous`를 `"-"` 로 적어도 됩니다.

### 3-2. 새 캘린더 일정 추가하기

`data.js` 맨 아래쪽 `const calendarEvents = [` 부분을 찾아서 아래처럼 한 줄 추가하면 됩니다. `indicatorId`는 위에서 만든 지표의 `id`와 반드시 똑같아야 캘린더에서 클릭했을 때 상세 설명이 연결됩니다.

```js
{ date: "2026-08-15", time: "09:00 (KST)", timeKST: "09:00", indicatorId: "kr_new_indicator" },
```

- `date`는 `YYYY-MM-DD` 형식 (예: 2026-08-15)
- `time`은 발표국 현지시각, `timeKST`는 한국시각 — 참고용으로 자유롭게 적으면 됩니다.

## 4. 나중에 외부에 공유하고 싶을 때

지금은 순수 HTML/CSS/JS 파일로만 되어 있어서, 별도 설치 없이 폴더를 그대로 인터넷에 올리기만 하면 다른 사람도 접속할 수 있습니다. 준비되면 아래 두 방법 중 하나를 시도해 보세요.

**방법 A. GitHub Pages (무료, 계정 필요)**
1. github.com 가입 → 새 저장소(Repository) 생성
2. `economic-dashboard` 폴더 안의 파일들을 그 저장소에 업로드
3. 저장소 설정(Settings) → Pages 메뉴에서 배포 브랜치를 선택하면, 몇 분 뒤 `https://내아이디.github.io/저장소이름` 주소로 접속 가능

**방법 B. Netlify Drop (계정 없이 가장 빠름)**
1. app.netlify.com/drop 접속
2. `economic-dashboard` 폴더를 화면에 그대로 드래그&드롭
3. 바로 임시 주소가 생성됨 (계정을 만들면 주소 고정 및 재배포 관리 가능)

이 단계는 지금 당장 하지 않아도 되며, 필요할 때 다시 요청해 주시면 함께 진행할 수 있습니다.
